## 要求

### 脚本功能
1. 脚本必须能在 `{repo_path}` 目录下直接运行
2. 运行所有可执行的测试（不仅仅是单个测试）
3. 最终输出一个 `result.json` 文件到仓库根目录

### result.json 格式
```json
{{
    "passed": <int>,
    "failed": <int>,
    "error": <int>,
    "total": <int>,
    "tests": [
        {{"name": "test_name", "status": "pass|fail|error", "message": ""}}
    ]
}}
```

### 脚本要求
- 使用 `#!/bin/bash` 开头
- 设置 `set -e` 只在关键步骤（不要在测试运行步骤，因为失败的测试不应中断脚本）
- 安装必要的依赖（如 `pip install cocotb` 如果需要）
- 处理编译错误和运行错误
- 即使部分测试失败，也要继续运行其他测试并汇总结果
- 最后一定要生成 `result.json`
- 超时保护：单个测试不应超过 {test_timeout} 秒

### 关键注意事项
- 如果仓库使用 cocotb，确保设置正确的 `SIM` 变量（优先使用 icarus）
- 如果仓库使用 Makefile，优先使用现有的 make targets
- 如果需要编译，确保所有源文件路径正确
- 不要 hardcode 绝对路径，使用相对路径或 `$PWD`
