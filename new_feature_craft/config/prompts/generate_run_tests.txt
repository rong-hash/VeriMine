## 要求

### 脚本功能
1. 脚本必须能在 `{repo_path}` 目录下直接运行
2. 运行所有可执行的测试（不仅仅是单个测试）
3. 最终输出一个 `result.json` 文件到仓库根目录

### result.json 格式
```json
{{
    "passed": <int>,
    "failed": <int>,
    "error": <int>,
    "total": <int>,
    "tests": [
        {{"name": "test_name", "status": "pass|fail|error", "message": ""}}
    ]
}}
```

### 脚本要求
- 使用 `#!/bin/bash` 开头
- **不要在测试运行部分使用 `set -e`**，因为失败的测试不应中断脚本
- 安装必要的依赖（如 `pip install cocotb` 如果需要）
- 处理编译错误和运行错误
- 即使部分测试失败，也要继续运行其他测试并汇总结果
- 最后一定要生成 `result.json`

### 超时保护
- 使用 `timeout` 命令为每个仿真添加超时保护，超时时间设为 **3 小时**（10800 秒）
  ```bash
  timeout 10800 vvp sim.vvp
  timeout 10800 make SIM=icarus
  ```
- 如果 `timeout` 返回 exit code 124，标记该测试为 FAIL（原因：simulation timeout）
- 注意：Phase 1 已经检查并修复了 testbench 中缺失的 `$finish` 终止信号，正常情况下仿真应该能自行终止。这个 3 小时超时只是最后的安全网。

### Bash 脚本编写注意事项
- 使用 `[[ ]]` 时确保变量被正确引用：`[[ "$var" -eq 0 ]]`
- 不要在 `[[ ]]` 里使用含换行的变量
- 使用 `$(())` 进行算术运算，不要用 `expr`
- 字符串比较用 `==`，数字比较用 `-eq`
- 确保所有变量在使用前已初始化

### 关键注意事项
- 如果仓库使用 cocotb，确保设置正确的 `SIM` 变量（优先使用 icarus）
- 如果仓库使用 Makefile，优先使用现有的 make targets
- 如果需要编译，确保所有源文件路径正确
- 不要 hardcode 绝对路径，使用相对路径或 `$PWD`
